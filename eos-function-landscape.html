The 1-D landscape trained with preconditioned GD: x ← x − η·P·∇L. plain GD: P = I — the baseline; every chart reduces to the ordinary quantities. signed GD: P = 1/|g| → constant step size η. spectral GD: P from kron((GGᵀ)−¼, (GᵀG)−¼) per layer — in 1-D both reduce to sign(g), so signed ≡ spectral here. Model  ·  fᵢ(w) = −cᵢ·eb·w

n datapoints, one parameter. Each sample has its own sensitivity cᵢ = 1 + σ·zᵢ with zᵢ spread over [−1, 1]; every target is the same y. Because one parameter cannot match n spread targets, the residual splits permanently into a fittable part inside range(J) — the only direction the optimizer can move outputs along — and an orthogonal part no optimizer can ever remove. σ = 0 makes all samples identical and collapses the problem back to a single datapoint. SGD: set B < n and each step draws a fresh uniform batch of B samples (without replacement) and steps on that batch's gradient; B = n is full-batch. Every chart still reports full-sample quantities, so the loss, the sharpness and the 2/η line stay comparable across batch sizes — only the update is stochastic. The seed makes a run reproducible; change it to see a different noise realisation.
n: samples · B: batch · σ: sample spread · b: curvature growth · y: target · η, T: optimizer.

Model  ·  fᵢ = a·eb·w₂·w₁·cᵢ + μ·w₂

n datapoints, two parameters. The oscillating output enters weighted by cᵢ = 1 + σ·zᵢ (spread across samples) while the fitting pull μ·w₂ moves every output equally — so in function space the oscillation direction and the fitting direction are different vectors whenever σ ≠ 0, and coincide exactly when σ = 0 (the single-datapoint degeneracy). Targets are yᵢ = y₂ + τ·(zᵢ² − ⅓): the Jacobian rows span {1, c} = {1, z}, so the quadratic part τ·zᵢ² is what the two parameters can never reach — set τ = 0 and the targets become exactly reachable and ‖r⊥‖ collapses to zero. With n > 1 the oscillation also re-centres: it happens around w₁* ≠ 0 rather than around the valley, so watch x_t rather than the sign of w₁. SGD: with B < n each step draws a fresh uniform batch of B samples and steps on that batch's gradient (the preconditioner is built from the same batch); B = n is full-batch. Charts report full-sample quantities so the 2/η comparison stays meaningful; the seed fixes the noise realisation.
n: samples · B: batch · σ: spread · a, b: valley sharpness and its gradient · μ: fitting strength · y₂: target · η, T: optimizer.

Model  ·  fᵢ = (a + b·w₂)·cᵢ·w₁ + μ·w₂

Constant function Hessian. Same tilted-valley story as the previous tab, but the exponential gain a·eb·w₂ is replaced by the linear gain a + b·w₂, which makes each fᵢ bilinear in (w₁, w₂). So every ∇²fᵢ = [[0, b·cᵢ],[b·cᵢ, 0]] is constant and ∇³fᵢ ≡ 0 everywhere — the third-derivative term of ∇S vanishes identically. Yet the full picture survives: sharpness still grows (H₁₁ = 2(a+b·w₂)²·mean c² rises as the fit pulls w₂ up), the iterate still oscillates, and the rectified w₁² coupling still pushes it back out — proving self-stabilization here rides entirely on the J·Q term (Jacobian × function-Hessian), not on ∇³f. Batch B < n gives SGD; τ sets the unfittable target curvature; the linearise switch removes the rectification.
a: base gain · b: gain slope (= off-diagonal of ∇²f) · n: samples · B: batch · σ: spread · μ: fitting strength · y₂: target · η, T: optimizer.
